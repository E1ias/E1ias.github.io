spark(1)

spark的基本概念
● RDD：是弹性分布式数据集（Resilient Distributed Dataset）的简称，是分布式内存的一个抽 象概念，提供了一种高度受限的共享内存模型；
● DAG：是Directed Acyclic Graph（有向无环图）的简称，反映RDD之间的依赖关系；
● Executor：是运行在工作节点（Worker Node）上的一个进程，负责运行任务，并为应用程序存储数据；
● 应用：用户编写的Spark应用程序；
● 任务：运行在Executor上的工作单元；
● 作业：一个作业包含多个RDD及作用于相应RDD上的各种操作；
● 阶段：是作业的基本调度单位，一个作业会分为多组任务，每组任务被称为“阶段”，或者也被 称为“任务集”。

开启spark-shell
cd到spark主目录 运行 ./bin/spark-shell

开启HDFS服务
cd HADOOPPATH
./sbin/start-dfs.sh
./sbin/start-dfs.sh dfs -mkdir -p /user/hadoop 为当前登录用户创建目录
./bin/hdfs dfs -ls /user/hadoop 查看HDFS文件系统中的目录和文件
./bin/hdfs dfs hdfs下执行任何shell语句时前面都要加上前面这一条
RDDs支持两种类型的操作
● actions 在数据集上运行计算后返回值
● transformations 转换操作，从现有数据集创建一个新的数据集（Lazy，action触发）

python脚本提交
./bin/spark-submit examples/src/main/python/pi.py

上下文包括
● Spark context available as sc.（构建运行环境，由任务控制节点Driver创建）
● SQL context available as sqlContext.

聚类算法(1)

Rodriguez A, Laio A. Clustering by fast search and find of density peaks[J]. Science, 2014, 344(6191): 1492-1496.

该聚类算法基于以下假设

  • 聚类中心被低密度的邻点包围
  • 聚类中心距离其他密度大的点足够远

首先对于数据集中每个数据点$i$，我们计算两个量

  • $\rho_i$ 局部密度
    $\rho_i =\sumj \sigma (d{ij}-d_c)$

其中$\sigma_i =(x<0)?1:0$, $d_c$是截断距离，是一个超参数。$\rho_i$其实就是点$i$在$d_c$范围内的邻居点个数。

  • $\delta_i$ 到比其密度高的点的最小距离

  • $\deltai = max(d{ij})$,当i为密度最大的点时

  • $\deltai =min(d{ij})$，j为密度比i大的点

选取聚类中心
$\gamma_i=\rho_i\delta_i$
把$\gamma$降序排列，画图取前若干个作为聚类中心。
